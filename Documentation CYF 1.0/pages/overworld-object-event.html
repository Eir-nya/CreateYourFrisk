The Overworld: The Event Object




Event.Teleport(string name, number dirX, number dirY) Teleports the given event to a given point in space. The position (dirX and dirY) is computed based off of the bottom left corner of the map.


Event.MoveToPoint(string name, number dirX, number dirY, bool wallPass = false, bool waitEnd = true) Moves an event to a given point in space. The event's speed will be the speed variable of its EventOW component, so choose that wisely.
Set wallPass to true to ignore any obstacle that the event may encounter while moving.
By default, the script will pause until the event has stopped moving. Set waitEnd to false to bypass this and directly continue the script instead.


Event.SetAnimHeader(string name, string anim) If the event given has a CYFAnimator component, it'll set anim as its anim header.
Don't forget that if the anim header is an existing animation, that animation will be launched instead of the component's normal behaviour.


Event.GetAnimHeader(string name) Returns the anim header of the given event, if it has a CYFAnimator event.


Event.SetDirection(string name, int dir) If the event has a CYFAnimator component, it sets the direction of the event.
2 for Down, 4 for Left, 6 for Right and 8 for Up.


Event.Rotate(string name, float rotateX, float rotateY, float rotateZ, bool anim = true, bool waitEnd = true) This function rotates the event. The Z rotation is the rotation we normally use, while X and Y rotations are 3D rotations.
To see some examples of rotation, remove the comment block of the function EventPage4 of Mods/Examples/Lua/Events/EventTest.lua.
Set anim to false to disable the animation of the rotation.
By default, the script will pause until the event has stopped rotating. Set waitEnd to false to bypass this and directly continue the script instead.


Event.Stop() If this function is launched in a normal event, it stops the event instantly. If it's launched in a coroutine event, it'll launch Event.StopCoroutine() instead. If you want to stop the coroutine instantly, just add return after this function.


Event.StopCoroutine(string eventName = "thisevent") Stops the coroutine of the given event. Use the string thisevent to stop the coroutine of this event.


Event.Remove(string name) Remove the given event from the scene. It won't be accessible afterwards until the player exits the scene! However, if the player comes back, it'll be here again.
To permanently remove an event, set its page to -1.


Event.SetPage(string name, int page) Sets the page of the given event. Set it to -1 to totally remove an event.


Event.GetPage(string name) Returns the current page of the given event.


Event.GetSprite(string name) Returns the sprite object of the given event.


Event.CenterOnCamera(string name, int speed = 5, bool straightLine = false, bool waitEnd = true) Centers the camera on the given event. The camera's speed will be speed pixels per second in each direction, unless you set straightLine to true. In that case, the camera will follow a line to the center of the event.
By default, the script will halt execution until the camera has stopped moving. Set waitEnd to false to bypass this and directly continue the script instead.


Event.GetName() Returns the name of the event which is launching this function.


Event.GetPosition(string name) Returns a table of two values containing the position of the given event.


Event.IgnoreCollision(string name, bool ignore) Sets the collision mode of the event. If ignore is set to true, all other events may pass through it normally.
Events launched when the player touches them need their collision to be ignored in order to function properly.


Event.SetSpeed(string name, float speed) Sets the speed of an event.
